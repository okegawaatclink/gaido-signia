## 確定要件

### システム概要

電子書籍に著者の手書きサインを合成して、ファンに個別に提供するSaaSシステム。

### 機能要件

#### 管理側（管理画面）

| # | 機能 | 説明 |
|---|------|------|
| M1 | 電子書籍登録 | PDF/EPUBファイルのアップロード・メタデータ登録（1著者あたり最大50冊、1ファイル50MBまで） |
| M2 | サイン登録 | タブレット上でCanvas手書きによるサインデータ登録。共通サインと個別サイン（宛名付き）の両方に対応 |
| M3 | サイン合成 | サイン画像を電子書籍に合成（専用サインページとして挿入）。リアルタイム（数秒以内）で処理完了 |
| M4 | 著者アカウント管理 | システム管理者が著者アカウントを作成・管理 |
| M5 | 書籍一覧・管理 | 登録済み書籍の一覧表示・編集・削除 |

#### 外部システム連携API（このシステムが提供）

| # | API | 説明 |
|---|-----|------|
| A1 | 電子書籍ーユーザー紐づけAPI | 外部購入システムから呼び出し、特定のファンに書籍アクセス権を付与 |
| A2 | サインデータ登録API | 外部システムから著者のサインデータを登録 |

#### クライアント側（ファン向け）

| # | 機能 | 説明 |
|---|------|------|
| C1 | ソーシャルログイン | Google / Apple IDでのログイン |
| C2 | 本棚（シンプルなリスト） | 自分に紐づいたサイン入り電子書籍の一覧表示 |
| C3 | 電子書籍閲覧 | ブラウザ上でのPDF/EPUBビューアー（DRM保護付き、ダウンロード不可） |

#### やらないこと（スコープ外）

- 決済・購入機能
- メール/プッシュ通知
- サインの真正性検証（電子署名・デジタル証明書）
- 多言語対応（日本語のみ）

### 非機能要件

| カテゴリ | 要件 |
|---------|------|
| ユーザー規模 | 中規模（～1,000人） |
| パフォーマンス | サイン合成処理: リアルタイム（数秒以内）。API応答: p95で500ms以下 |
| 可用性 | 99.5%以上（SaaS提供のため） |
| セキュリティ | OWASP Top 10対策、ソーシャルログイン（OAuth 2.0/OIDC）、DRMによるコンテンツ保護 |
| 監査ログ | 基本的なログ（ログイン/ログアウト、重要操作の記録） |
| 対応デバイス | PC（Chrome, Firefox, Safari, Edge）、モバイル（iOS Safari, Android Chrome）、タブレット（iPad Safari - 管理側サイン手書き用） |
| 言語 | 日本語のみ |
| ファイルサイズ | 電子書籍: 最大50MB/ファイル |
| ストレージ | クラウドストレージ（S3互換） |
| 配布形態 | SaaS提供（AGPLライセンスのライブラリは使用しない） |

### ロール定義

| ロール | 説明 |
|--------|------|
| システム管理者 | 著者アカウント作成・システム全体管理 |
| 著者 | 電子書籍登録・サイン登録・サイン合成 |
| ファン（クライアント） | ソーシャルログイン・本棚閲覧・電子書籍閲覧 |

### ユーザーフロー

1. **著者フロー**: システム管理者がアカウント作成 → 著者が電子書籍をアップロード → サインを手書き登録 → ファンに個別/共通サインを付与
2. **ファンフロー**: 外部システムで書籍を購入 → 外部システムがAPI経由でユーザーと書籍を紐づけ → ファンがソーシャルログイン → 本棚でサイン入り書籍を閲覧
3. **サイン合成フロー**: 著者が個別サイン（宛名付き）または共通サインを選択 → サイン画像を電子書籍の専用ページとして挿入 → 合成完了（数秒以内）

## 専門家分析

### PO分析

- **MVP範囲**: 電子書籍登録・サイン登録・合成・閲覧の基本フローを最優先。通知・真正性検証・多言語は後回し
- **ビジネス価値**: 著者とファンの特別な繋がりを電子書籍で実現。物理的なサイン会の代替/補完
- **コスト意識**: 中規模SaaSとして、クラウドコストを抑えつつスケーラブルな設計

### Architect分析

- **技術選定**: Next.js（フロントエンド）+ Express.js（バックエンドAPI）+ PostgreSQL
  - Next.js: SSR/SSGによるSEO対応、React Canvasでタブレット手書き対応、MIT License
  - Express.js: 外部APIエンドポイント提供に適した軽量フレームワーク、MIT License
  - PostgreSQL: リレーショナルデータ管理、JSONB対応、OSS（PostgreSQL License）
- **サイン合成方式**: 専用サインページの挿入方式を採用
  - PDF: pdf-lib（MIT License）でサインページを生成しPDFに挿入
  - EPUB: HTMLページとしてサインページを生成しEPUBに挿入
  - 理由: オーバーレイ方式よりも実装がシンプルで、PDF/EPUB両方に統一的に対応可能
- **DRM方式**: サーバーサイドレンダリング型ビューアー
  - ブラウザ上でpdf.js/epub.jsを使用してレンダリング
  - コンテンツは署名付きURL（有効期限付き）で配信
  - ダウンロードボタンの無効化、右クリック禁止、印刷制限
- **ファイルストレージ**: S3互換ストレージ（開発時はMinIO）
- **OSSライセンス注意**: SaaS提供のためAGPLライブラリは使用禁止

### QA分析

- **テスト戦略**: 
  - 当たり前品質（認証・認可・ファイルアップロード）: エッジケース網羅、自動化必須
  - 一元的品質（サイン合成・閲覧体験）: 中程度テスト、自動化推奨
  - 魅力的品質（UIアニメーション等）: 正常系中心
- **重点テスト項目**: サイン合成の品質（画像解像度・位置精度）、PDFとEPUBの両形式対応、タブレットでの手書き入力精度
- **対応ブラウザ**: Chrome最新, Firefox最新, Safari最新, Edge最新, iOS Safari, Android Chrome

### Security分析

- **認証**: OAuth 2.0 / OIDC（Google, Apple）。管理側はメールアドレス+パスワード+MFA推奨
- **認可**: ロールベースアクセス制御（RBAC）。ファンは自分の書籍のみアクセス可能
- **データ保護**: 電子書籍ファイルは暗号化して保存。署名付きURLで時間制限アクセス
- **OWASP対策**: 入力バリデーション、CSRF保護、XSS対策、SQLインジェクション防止
- **監査ログ**: ログイン/ログアウト、書籍アクセス、サイン操作、管理操作を記録
- **脅威**: コンテンツの不正コピー（DRM）、認証バイパス（OAuth）、APIの不正利用（レート制限・API Key認証）
