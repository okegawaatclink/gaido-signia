# devops.md 出力仕様

ソース解析フェーズが `ai_generated/intermediate_files/from_source/devops.md` に出力する際のセクション構造と記載ルール。

## セクション構造

devops.mdは以下のセクション構造で出力する。該当する情報がないセクションは省略可。

### 1. ビルド・起動手順
- ビルドコマンド、起動コマンド
- 依存パッケージのインストール手順
- 作業ディレクトリ

### 2. デプロイ構成
- 本番/ステージング環境への配置方法（AWS、GCP、Kubernetes等）
- デプロイスクリプト、Infrastructure as Code（Terraform等）
- ローカル開発でのdocker-compose起動方法はビルド・起動手順の範囲であり、デプロイ構成には含めない
- 本番デプロイの仕組みが存在しない場合はセクション自体を省略する

### 3. CI/CD設定
- CI/CDツール（GitHub Actions、CircleCI等）
- パイプラインの構成（ビルド→テスト→デプロイ）
- 自動化されている処理の一覧

### 4. 環境変数
- 環境変数の一覧と用途
- 環境ごとの設定ファイル（.env.development, .env.production等）
- シークレット管理方式

### 5. テスト実行方法
- テストコマンド
- テストフレームワーク
- カバレッジ取得方法

## 記載ルール

- 既存の `from_source/devops.md` がある場合（ローカル実行フェーズが先行出力済み）は、その内容を保持したうえで、ソース解析で得た情報を追記する
- ローカル実行フェーズは「ビルド・起動手順」「環境変数」セクションを出力するため、既存セクションの内容を保持しつつ追加情報をマージする
- ローカル実行フェーズが未出力のセクション（デプロイ構成、CI/CD設定、テスト実行方法等）は新規セクションとして追加する
- ソースコードから読み取れた事実のみを記載すること。推測は「推測:」と明記する
