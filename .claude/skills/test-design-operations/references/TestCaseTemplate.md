# テストケースIssue作成ルール

## Issue形式

```markdown
## テストケース: [ユーザーストーリーの要約]

### 元のユーザーストーリー
[PBI Issueのユーザーストーリーをそのまま引用]

### テスト手順
1. [前提条件を満たす]
2. [操作を行う]
3. [結果を確認する]

### 期待結果
- [期待される結果1]
- [期待される結果2]

### 対象PBI
- #[PBI番号]
```

## ラベル

`test-case` ラベルを付与すること（既存のEpic/PBI/Task階層とは独立）。

```bash
# テストケースIssue作成例
gh issue create --title "[テストケース] ログインしてダッシュボードを表示できる" \
  --body "..." \
  --label "test-case"
```

## 追加テストケースの提案

PBIのユーザーストーリーをIssue化した後、以下の観点で追加テストケースを提案すること：

1. **エラーケース**: 異常系の動作確認
2. **境界値**: 入力値の境界条件
3. **権限**: 認証・認可の動作確認
4. **パフォーマンス**: 大量データでの動作確認（必要に応じて）

## 注意事項

- 1つのユーザーストーリーにつき1つのテストケースIssueを作成
- テストケースIssueは既存のEpic/PBI/Task階層とは独立して管理
