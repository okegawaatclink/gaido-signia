## テンプレート（ここから下を出力する）

このPowerPointスライドのデザインを以下の仕様に合わせて仕上げてください。

### デザイン仕様

| 項目 | 値 |
|---|---|
| スライド背景色 | `{bg_slide: 例 #0f172a}` |
| タイトル文字色 | `{primary: 例 #ffffff}` |
| アクセントカラー | `{accent: 例 #3b82f6}` |
| 暗い背景上の文字色 | `{text_on_dark: 例 #ffffff}` |
| 明るい背景上の文字色 | `{text_dark: 例 #1a202c}` |
| 使用フォント | `{font: 例 游ゴシック}` |

### フォントサイズ基準（最小値。詳細推奨範囲は `references/SlideTheme.md` の `FONT_SIZE` を参照）

| 要素 | 最小サイズ |
|---|---|
| スライドタイトル | 34pt 以上 |
| セクション見出し | 26pt 以上 |
| 本文 | 20pt 以上 |
| テーブルセル | 16pt 以上 |

**フォントサイズ調整の原則（重要）:**

フォントサイズを変更する場合は、**スライド内の全テキスト要素の比率を維持したまま拡縮すること**。全要素を同一サイズに揃えてはならない。

- 修正前にスライド内の最大・最小フォントサイズを確認し、その比率を記録する
- 最小値が基準を下回っている場合は、比率を保ったまま全体をスケールアップする
- 例: タイトル28pt・本文16pt・キャプション12pt（比率 7:4:3）→ 本文20pt基準でスケールアップ → タイトル35pt・本文20pt・キャプション15pt（比率 7:4:3 を維持）
- タイトル・見出し・本文・キャプションなど、意図的に異なるサイズが設定されているものを同一サイズにしてはならない

### チェック・修正してほしい内容

1. **背景色の統一**: すべてのスライドの背景が上記の背景色になっているか確認し、白いままのスライドがあれば修正してください
2. **コントラストの確保（項目1の背景色変更後に必ず再確認）**: 以下の2段階で確認・修正してください
   - **最優先（コントラスト比1.0〜1.1）**: 白文字（`#FFFFFF`）が `#F8FAFC` / `#EBF4FF` / `#FFF5F5` 等の淡い背景色の上に配置されているシェイプは、テキスト色を濃い色（`#333333` または `#1a202c`）に変更してください
   - **WCAG AA基準（コントラスト比4.5未満）**: 背景色を暗くするかテキスト色を変更して4.5以上を確保してください。特に `#4299E1`（青）/ `#718096`（グレー）/ `#38A169`（緑）/ `#63B3ED`（薄い青）の背景上の白文字は要注意です
3. **フォントの統一**: 全スライドのフォントが `{font}` に統一されているか確認し、異なるフォントがあれば修正してください
4. **フォントサイズ**: 上記基準を下回っているテキストがあれば、上記「フォントサイズ調整の原則」に従い、スライド内の比率を維持したまま全体をスケールアップして修正してください
5. **レイアウトの整合性**: タイトル位置・余白・ボックスの配置がスライド間で統一されているか確認してください
6. **テーブルのスタイル**: ヘッダー行の背景色が `{accent}` になっているか確認してください
7. **フッター（コピーライト＋ページ番号）**: 表紙を除く全スライドのフッターに以下が揃っているか確認してください
   - 左下: `© ITOCHU Techno-Solutions Corporation`（9pt、グレー）
   - 右下: `{現在ページ} / {総ページ数}`（例: `2 / 33`、9〜10pt、グレー）
   - フッターがスライド下端ぎりぎりの場合は、下端から最低10pt以上の余白を確保してください
8. **タグ/バッジ/ラベル形式の要素（最優先）**: 短いラベルテキスト（目安: 10文字以下）を含む小さなボックス・角丸ボックス・ピル形状の要素は、テキストが**必ず1行**に収まっていなければなりません。以下を確認・修正してください
   - **検出方法**: テキストが2行以上に折り返されているにもかかわらず、ラベルとして機能している（他の同種要素と横並びになっている、背景色付きの小さいボックスである）場合はタグ/バッジ形式と判断してください
   - **修正手順**:
     1. そのシェイプの `word_wrap` を `False` に設定する
     2. シェイプの幅を「テキスト幅 + 左右内側余白各6pt以上」になるよう拡大する
     3. シェイプの高さをテキスト高さ + 上下内側余白各4pt以上になるよう調整する
     4. 同種のタグ/バッジが複数ある場合（事業領域タグ、スキルタグ等）は、**全要素を同じ高さ**に揃え、幅はテキスト量に応じて個別に調整する
     5. 隣接する要素との間隔が詰まりすぎた場合は、間隔を均等に再配置する
9. **テキストの図形はみ出し・不自然な改行**: テキストが親図形からはみ出している、またはテキストボックス幅が足りず不自然に改行されているものを修正してください。以下の優先順で対応してください
   1. テキストボックスのサイズを親図形の内側に収まるよう調整する（親図形の内側余白4pt以上を確保）
   2. それでも収まらない場合はフォントサイズを比率を維持したまま縮小する
   3. それでも収まらない場合は親図形のサイズを拡大する（ただし拡大後も隣接要素との間に8pt以上の余白を維持すること）
   - `wrap` が `none` に設定されているテキストボックスは `square` に変更した上でボックスサイズを調整すること（`none` のままではテキストのはみ出しが不可視になるだけで解消されない）。ただし項目8のタグ/バッジ形式の要素は除く
10. **スライド境界外への要素のはみ出し**: 要素の右端または下端がスライドの境界（`presentation.slide_width` / `presentation.slide_height`）を超えている場合、要素をスライド内に収まるよう移動またはサイズを縮小してください
    - **検出方法**: `shape.left + shape.width > prs.slide_width` または `shape.top + shape.height > prs.slide_height` の場合
    - **修正手順**: まず位置を移動してスライド内に収め、それでも収まらない場合はシェイプをスライド幅/高さに合わせて縮小する
11. **文字・図形の重なり**: テキストボックス・ボックス・図形同士が重なっているスライドがあれば修正してください。各要素の位置・サイズを調整し、隣接要素との間に最低8pt以上の余白を確保してください（項目9の手順③で親図形を拡大した場合も同様に確認すること）
    - **テーブルの上にテキストボックスが重なっている場合**: テキストボックスをテーブルの外側（上・下・左・右）に移動してください。テーブルの行・列を増やしてテキストを収容できる場合はそちらを優先してください
12. **テーブルの余白**: テーブルのセル内余白（上下左右）が詰まりすぎていないか確認してください。テキストがセル枠にくっついている場合は、セル内の余白（マージン）を上下4pt・左右6pt以上に広げてください
13. **テーブルの視認性**: 行区切り線・交互背景色がないテーブルは行の追跡が困難です。奇数行にスライド背景色より少し濃い薄いグレーの背景色を設定し、行間に区切り線を追加してください。列幅は内容量に応じて調整してください
14. **余白の活用（スライド全体を使った構成）**: コンテンツが一部に密集し、まとまった余白（縦・横問わず）が残っているスライドがあれば、以下の方針でスライド全体を使った構成に再配置してください
    - 要素全体をスライドの有効領域（タイトル下〜フッター上）に均等に分散させる
    - 各要素のサイズ（幅・高さ）をスライド領域に合わせて拡大し、余白を埋める
    - 複数列・複数行の構成の場合は、列間・行間の余白を均等にする
    - 要素の拡大によってテキストが窮屈になる場合は、内側の余白も比例して広げる
    - ただし要素のアスペクト比（縦横比）や相互の大きさの比率は維持すること
15. **凡例記号の統一**: スライド間で凡例の矢印・記号スタイル（▶ / ● / ▼ 等）が混在している場合は、全スライドで同一の記号に統一してください
16. **行間（lnSpc）の調整**: 行間が150%以上に設定されているテキストボックスがあれば120%以下に調整してください。行間が広すぎるとテキストボックスからテキストがはみ出す原因になります
    - **検出方法**: `paragraph.line_spacing` が `1.5` 以上（150%相当）のパラグラフを全スライドで検索する
    - **修正手順**: `paragraph.line_spacing = 1.2`（120%）に変更する。コンテンツ量が多いスライドではさらに `1.1` や `1.15` まで縮めて調整すること
    - 行間調整後に項目11（文字・図形の重なり）を再確認すること
