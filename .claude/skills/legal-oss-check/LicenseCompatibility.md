# ライセンス互換性マトリクス

OSSライセンスの互換性判定に使用するリファレンス。プロジェクトのライセンスと依存パッケージのライセンスの組み合わせに基づき、互換性を判定する。

> このマトリクスは一般的な理解に基づく参考情報です。ライセンスの法的解釈は個別の状況により異なるため、重要な判断には専門家の確認を推奨します。

## ライセンスカテゴリ

### Permissive（寛容型）

制約が少なく、ほとんどのプロジェクトで利用可能なライセンス。

| ライセンス | 主な条件 |
|-----------|---------|
| MIT | 著作権表示・ライセンス文の保持 |
| BSD-2-Clause | 著作権表示・ライセンス文の保持 |
| BSD-3-Clause | 著作権表示・ライセンス文の保持・名称使用制限 |
| Apache-2.0 | 著作権表示・ライセンス文の保持・変更点の明示・特許権の付与 |
| ISC | 著作権表示・ライセンス文の保持 |
| Unlicense | パブリックドメイン相当 |
| CC0-1.0 | パブリックドメイン相当 |

### Weak Copyleft（弱コピーレフト）

ライブラリとしてリンクする場合は独自ライセンスでの配布が可能。改変した場合はソース公開が必要。

| ライセンス | 主な条件 |
|-----------|---------|
| LGPL-2.1 | ライブラリ改変時はソース公開。リンクのみの場合は独自ライセンス可 |
| LGPL-3.0 | LGPL-2.1と同様 + リバースエンジニアリングの許可 |
| MPL-2.0 | ファイル単位のコピーレフト。改変ファイルのみソース公開 |
| EPL-2.0 | モジュール単位のコピーレフト |

### Strong Copyleft（強コピーレフト）

派生物全体に同じライセンスの適用が求められるライセンス。

| ライセンス | 主な条件 |
|-----------|---------|
| GPL-2.0 | 派生物全体にGPL-2.0の適用が必要 |
| GPL-3.0 | 派生物全体にGPL-3.0の適用が必要 + 特許権・DRM制限 |
| AGPL-3.0 | GPL-3.0と同様 + ネットワーク経由の利用もソース公開義務 |

### Network Copyleft

| ライセンス | 主な条件 |
|-----------|---------|
| AGPL-3.0 | SaaSとして提供する場合もソースコード公開が必要 |
| SSPL | MongoDB等で使用。SaaS提供時にサービス全体のソース公開が必要 |

## 互換性マトリクス（プロジェクトライセンス × 依存ライセンス）

### プロジェクトがPermissive（MIT, Apache-2.0等）の場合

| 依存パッケージのライセンス | 判定 | 備考 |
|-------------------------|------|------|
| MIT, BSD, ISC, Unlicense | GREEN | 互換性あり |
| Apache-2.0 | GREEN | 互換性あり（MIT→Apache含む場合の特許条項に注意） |
| MPL-2.0 | YELLOW | ファイル単位の義務。利用方法による |
| LGPL-2.1/3.0 | YELLOW | ライブラリとしてリンクのみなら可。改変は注意 |
| GPL-2.0/3.0 | RED | プロジェクト全体にGPLの適用が必要になる可能性 |
| AGPL-3.0 | RED | ネットワーク利用でもソース公開義務の可能性 |
| SSPL | RED | SaaS提供時にサービス全体のソース公開義務の可能性 |

### プロジェクトがGPL-3.0の場合

| 依存パッケージのライセンス | 判定 | 備考 |
|-------------------------|------|------|
| MIT, BSD, ISC, Unlicense | GREEN | 互換性あり |
| Apache-2.0 | GREEN | GPL-3.0と互換 |
| LGPL-2.1/3.0 | GREEN | 互換性あり |
| MPL-2.0 | GREEN | 互換性あり（MPL-2.0はGPL互換を明示） |
| GPL-2.0 | YELLOW | 「GPL-2.0 only」の場合は非互換。「GPL-2.0-or-later」なら互換 |
| GPL-3.0 | GREEN | 同一ライセンス |
| AGPL-3.0 | RED | AGPLの追加義務が発生する可能性 |

### プロジェクトが独自（プロプライエタリ）の場合

| 依存パッケージのライセンス | 判定 | 備考 |
|-------------------------|------|------|
| MIT, BSD, ISC, Unlicense | GREEN | 互換性あり |
| Apache-2.0 | GREEN | 互換性あり（著作権表示・NOTICE保持が必要） |
| MPL-2.0 | YELLOW | 改変ファイルのソース公開義務 |
| LGPL-2.1/3.0 | YELLOW | ライブラリとしてリンクのみなら可。静的リンクは注意 |
| GPL-2.0/3.0 | RED | プロプライエタリ配布と非互換の可能性が高い |
| AGPL-3.0 | RED | SaaS提供でもソース公開義務の可能性 |
| SSPL | RED | SaaS提供時にサービス全体のソース公開義務の可能性 |

## 配布形態による影響

| 配布形態 | コピーレフトの影響 |
|---------|------------------|
| OSSとして公開 | ライセンス条件に完全に従う必要あり |
| パッケージ配布（バイナリ） | 配布時にライセンス条件が適用される |
| SaaSとして提供 | 通常GPLは適用されないが、AGPLは適用される |
| 社内利用のみ | 配布に該当しないためコピーレフトは通常適用されない |

## 判定時の注意事項

- **ライセンスバージョン**: `GPL-2.0-only` と `GPL-2.0-or-later` は互換性が異なる
- **デュアルライセンス**: 複数ライセンスから選択可能な場合は、互換性の高い方を選択できる
- **例外条項**: 「Classpath Exception」等のライセンス例外がある場合は条件が変わる
- **UNKNOWN**: ライセンスが不明なパッケージは YELLOW とし、手動確認を推奨する
- **カスタムライセンス**: 標準ライセンスに該当しないものは YELLOW とし、個別確認を推奨する
