# リスク評価基準（RiskMatrix）

契約書レビュー時のリスク判定基準。各条項をGREEN/YELLOW/RED の3段階で評価する。

> この基準は一般的な目安です。業種・契約種別・当事者の立場によって適切な基準は異なります。Playbookに組織固有の基準がある場合はそちらを優先してください。

## 判定基準の概要

| レベル | 意味 | 出力時の表現例 |
|--------|------|--------------|
| GREEN | 一般的な基準の範囲内と考えられる | 「一般的な範囲内と考えられます」 |
| YELLOW | 確認・交渉を検討する余地がある可能性 | 「〜という観点から確認を検討する余地があるかもしれません」 |
| RED | 専門家への相談を推奨する事項 | 「専門家に相談されることを推奨します」 |

## 条項別の判定基準

### 秘密保持期間

| レベル | 基準 |
|--------|------|
| GREEN | 1〜3年の範囲 |
| YELLOW | 3〜5年、または期間の定めがない場合 |
| RED | 永久秘密保持義務、または秘密保持条項自体が存在しない |

### 責任制限・損害賠償

| レベル | 基準 |
|--------|------|
| GREEN | 損害賠償上限が契約金額の範囲内、間接損害免責あり |
| YELLOW | 損害賠償上限が契約金額の2倍以上、または上限の定めがない |
| RED | 無制限の損害賠償責任、または一方当事者のみ不利な条件 |

### 知的財産権

| レベル | 基準 |
|--------|------|
| GREEN | 成果物の権利帰属が明確、既存IPの取扱いが定められている |
| YELLOW | 権利帰属が曖昧、または一方的な譲渡条件がある |
| RED | 既存IPも含めた包括的権利譲渡、または知財条項自体が存在しない |

### データ保護

| レベル | 基準 |
|--------|------|
| GREEN | 個人情報保護法準拠の規定あり、セキュリティ対策の記載あり |
| YELLOW | データ保護規定はあるが具体性に欠ける |
| RED | データ保護規定なし、またはデータの無制限利用を許容する条項 |

### 解除条件

| レベル | 基準 |
|--------|------|
| GREEN | 双方に公平な解除条件、合理的な催告期間あり |
| YELLOW | 一方のみに有利な解除条件、催告なし解除の範囲が広い |
| RED | 一方的な即時解除権、解除時の清算規定なし |

### 競業避止

| レベル | 基準 |
|--------|------|
| GREEN | 期間1年以内、地域・業種の制限が合理的 |
| YELLOW | 期間1〜3年、または制限範囲が広い |
| RED | 期間3年超、制限範囲が不合理に広い、または違約金が過大 |

### 準拠法・管轄

| レベル | 基準 |
|--------|------|
| GREEN | 日本法準拠・国内裁判所管轄（日本企業同士の場合） |
| YELLOW | 海外法準拠、または管轄裁判所が遠方 |
| RED | 準拠法・管轄の定めなし、または著しく不利な法域 |

### 再委託

| レベル | 基準 |
|--------|------|
| GREEN | 再委託に事前承認が必要、再委託先の管理責任が明確 |
| YELLOW | 再委託が事後報告のみで可能 |
| RED | 再委託に関する規定なし |

## 総合リスク判定

- RED項目が**1件以上**ある場合: 総合判定 = RED（専門家相談を推奨）
- YELLOW項目が**3件以上**ある場合: 総合判定 = YELLOW
- 上記以外: 総合判定 = GREEN

## redline提案の方針

YELLOW/RED判定の条項に対して修正案テキストを提示する際の方針:

- 修正案は「参考案」であることを明記する
- 一般的に使用される条項表現をベースにする
- 当事者双方に公平な内容を目指す
- 業界標準や慣行を考慮した表現を使用する
- 法的な正確性は専門家の確認が必要である旨を付記する
