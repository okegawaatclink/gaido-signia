# 提案書デザインガイドライン

本ファイルは提案書作成時のデザインルールを定義する。
proposal-initの壁打ち時にこのガイドラインを参照し、
story.mdの「デザイン方針」セクションに案件ごとの具体値を記入する。

**重要**: このガイドラインはfrontend-designのSKILL.mdに直接渡さないこと。
story.mdの「デザイン方針」経由でfrontend-designに指示を伝える。

---

## レイアウト【最重要・絶対厳守】

- 各ページのヘッダー（見出し）は表紙などを除き全スライドで同じ位置に揃えること
- ヘッダーの位置・サイズ・フォントは全ページ統一
- 構成: 上部にヘッダー → 中央にメインコンテンツ → 下部に補足（あれば）
- 余白を恐れない: 詰め込みすぎず、適切なネガティブスペースを確保
- 16:9のページからはみ出さないこと

## タイポグラフィ

- フォント: Noto Sans JP（HTMLモック用デフォルト）
- 階層: 見出し用と本文用でウェイトを明確に分ける
- サイズ: 見出しは大胆に、本文は読みやすく
- 一貫性: 全スライドで同じタイポグラフィルールを徹底

> **PPTX生成時のフォント変換ルール**: HTMLでNoto Sans JPを指定していても、PPTXではWindows互換フォントに変換する。
> `Noto Sans JP` → `游ゴシック`（フォールバック: `メイリオ`）
> 理由: Noto Sans JPはWindows標準インストールされておらず、受け取り側PCで文字化け・レイアウト崩れが発生する。

## カラー

- ベース: 落ち着いた青系またはグレー系
- アクセント: 支配的なベースカラーに対して、シャープなアクセントカラーを1色設定
- 配分: 均等に色を使わず、ベース80% + アクセント20%程度のメリハリをつける
- NG: 紫グラデーション、レインボー配色など「AIっぽい」配色は避ける

## ビジュアル

- シャドウ: 使用しない
- 背景: 単色べた塗りだけでなく、微妙なグラデーションやテクスチャで深みを持たせてもよい
- 図表: シンプルで見やすいものを優先、装飾過多にしない
- トーン: ビジネス向け、信頼感と専門性を感じるデザイン

## 避けるべきこと

- ありきたりな配色やレイアウト
- 装飾過多・情報過多なスライド
- 統一感のない色使い・フォント使い
- 「よくあるAI生成スライド」感

---

## story.mdへの反映方法

壁打ち時に、上記ガイドラインをもとに案件ごとの具体値を決定し、
story.mdの「デザイン方針」セクションに以下の形式で記入する。

```markdown
## デザイン方針

| 項目 | 内容 |
|---|---|
| トーン | ビジネス・フォーマル |
| フォント | Noto Sans JP（見出し: Bold / 本文: Regular） |
| ベースカラー | #1a365d（ダークネイビー） |
| アクセントカラー | #2b6cb0（ブルー） |
| 背景 | ホワイト、薄いグレーのグラデーション |
| レイアウト | ヘッダー固定、16:9、余白重視 |
```

この情報が後工程（frontend-design、slide-generator）に引き継がれる。
