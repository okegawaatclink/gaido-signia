## 体制図スライドのレイアウト仕様

体制図（CTC側プロジェクト体制）を含むスライドは、以下のレイアウト・デザイン仕様に従うこと。

> **実装方法の使い分け（重要）**:
> - **HTMLモック生成時**（`/frontend-design`）: スライドの視覚的な参考として体制図をHTMLで表現する。デザイン確認用。
> - **PPTX生成時**（`/slide-generator`）: HTMLモックを画像化して貼り付けてはならない。必ず `tools/drawio_builder.py` の `OrgChartBuilder` で `.drawio` ファイルを作成し、PNGエクスポートして `add_picture()` で埋め込むこと。
> - 理由: python-pptxで体制図の複雑な接続線・ボックスをネイティブ再現するのは困難なため、draw.ioのPNG出力を使用する。

### デザイン仕様

| 要素 | 仕様 |
|------|------|
| スライド背景 | 薄い水色（`#d6eaf8` 相当） |
| ボックスヘッダー | ダークネイビー（`#1a5276`）、白文字、太字 |
| ボックスボディ | 白背景、黒文字 |
| ボックス形状 | 角丸（rounded）、swimlane形式（ヘッダー＋ボディ分離） |
| 接続線 | 直角エルボー（orthogonal）、ダークネイビー（`#1a5276`）、太さ2px |
| CTCロゴ | 左上に配置（`tools/ctc_logo.png` または類似画像） |
| フォントサイズ | ヘッダー: 11pt、ボディ: 10pt |

### 座標・サイズ仕様（1280×720px基準）

**重なりを防ぐため、以下の座標・サイズを厳守すること。**

ボックスの基本サイズ:

| 種別 | 幅 | 高さ |
|------|----|------|
| 標準ボックス | 190px | 85px |
| 大きめボックス（人数が多い場合） | 190px | 110px |

列ごとのx座標（左端基準）:

| 列 | x座標 | 幅 | 備考 |
|----|-------|----|------|
| 左列（技術系） | 30px | 190px | 右端: 220px |
| 中央左（PMの左配下） | 280px | 190px | 右端: 470px |
| 中央（PM） | 450px | 190px | 右端: 640px（中央配置） |
| 中央右（PMの右配下） | 620px | 190px | 右端: 810px |
| 右列（営業系） | 870px | 190px | 右端: 1060px |

**列間の最小間隔は60px以上確保すること。**

行ごとのy座標（上端基準）:

| 行 | y座標 | 備考 |
|----|-------|------|
| 第1行（統括責任者・PM） | 60px | |
| 第2行（支援・配下チーム） | 200px | 第1行との間隔: 55px以上 |
| 第3行（さらに配下） | 350px | 第2行との間隔: 55px以上 |
| フッター（© ITOCHU...） | 690px | |

**注意**: ボックスの高さが変わる場合、次の行のy座標も連動して下げること（前行のy座標 + 前行の高さ + 55px以上）。

### 3列の役割分担

| 列 | 内容 | 備考 |
|----|------|------|
| 左列 | 技術系ライン（技術統括責任者 → 技術支援） | 縦の直列階層 |
| 中央列 | PM → 配下チーム（プロジェクトサポート、構築作業等） | PMを頂点にしたツリー構造 |
| 右列 | 営業系ライン（営業統括責任者 → 営業管理者 → 担当営業） | 縦の直列階層 |

### プレースホルダー

実際の案件に合わせて以下を置換する：
- `所属部署` → 実際の部署名
- `XX XX` → 実際の人名（姓 名）
- 役職・担当チーム名 → 案件に応じて追加・削除可
