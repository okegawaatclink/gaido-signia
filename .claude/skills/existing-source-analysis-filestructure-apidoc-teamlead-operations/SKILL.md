# チームリード手順: ソースファイルコメント付与タスク管理

あなたはチームリードです。

> **最重要ルール: あなた自身はソースファイルへのコメント付与を絶対に行わないこと。Editツール・sedコマンド等でソースファイルを変更してはならない。コメント付与はすべてチームメンバー（Agent tool）に委任し、あなたはタスク管理（割り当て・検証・記録）のみを行う。**

## 処理手順

### 1. タスクファイル読み込み

`ai_generated/intermediate_files/from_source/progress/source_file_tasks.md` をReadツールで読み込む。

### 2. 未完了タスクの抽出

`- [ ]` の行を抽出し、**先頭から最大15件**をメンバーに割り当てる対象とする。

### 3. メンバー割り当て

Agent toolを1レスポンスで複数並列起動し、各メンバーに1ファイルずつ割り当てる。

メンバーへのプロンプト:
```
まず .claude/skills/existing-source-analysis-filestructure-apidoc-member-operations/SKILL.md をReadツールで読み込み、その手順に従ってコメント・ドキュメントを追加してください。

対象ファイル: {ファイルパス}
```

### 4. 完了検証（最重要）

メンバーから完了報告を受けたら、**1件ずつ直列に**以下の手順を実行する。複数メンバーの報告をまとめて処理してはならない。

**1件のファイルに対する処理（アトミックに実行）:**

1. grepで検証する:
```bash
grep -c 'AUTO_GENERATED' {ファイルパス}
```
2. 結果に応じて `source_file_tasks.md` を更新する:
   - **1件以上**: 該当行を `- [x]` に更新する
   - **0件**: `- [ ]` のまま残す。メンバーの失敗として報告に含める
3. 次のメンバーの報告に進む

**この3ステップを中断せず1ファイルずつ完了させること。** メンバーが「完了しました」と報告しても、grepで確認できなければ `[x]` にしてはならない。

### 5. 完了報告

以下を報告する:
- 処理したファイル数（成功/失敗）
- 失敗したファイルのリスト（もしあれば）
- `source_file_tasks.md` の残り未完了数

### 制約

- **15件処理したら、未完了タスクが残っていても完了を報告して終了する**
- チームリード自身はコメント付与作業を行わない
- `source_file_tasks.md` 以外のファイルを編集しない
