# レポートテンプレート

本ファイルは案件アドバイザーが生成するドキュメントのテンプレートを定義する。

## セクション構成の原則

全テンプレートは以下の順序でセクションを構成すること（結論ファースト）:

1. **ヘッダー**（案件名・日付・ステータス）
2. **判定結果**（結論を最初に提示）
3. **スコア詳細**（マークダウンテーブル形式）
4. **改善ヒント / 次のアクション / コピペ用文面**
5. **スコアリングデータ（JSON）**（末尾に `<details>` で折りたたんで配置）

スコアリングJSONは `ScoringCriteria.md` のフォーマットに従い、レポート末尾の `<details>` タグ内に配置する。人間が読むセクションの途中にJSONを挿入してはならない。

## 出力先
- 営業ヒアリングまとめ: `ai_generated/advisor/hearing_YYYYMMDD_HHMMSS.md`
- プロジェクトプロファイル（PPT用story.md）: `ai_generated/advisor/profile_YYYYMMDD_HHMMSS_story.md`
- プロジェクトプロファイル（PPTX）: `ai_generated/advisor/profile_YYYYMMDD_HHMMSS.pptx`
- 確定レポート: `ai_generated/advisor/report_YYYYMMDD_HHMMSS.md`
- 情報不足メモ・No Bidメモ: `ai_generated/advisor/memo_YYYYMMDD_HHMMSS.md`
- 簡易No Bidメモ: `ai_generated/advisor/nobid_YYYYMMDD_HHMMSS.md`

技術担当に確認すると、以下が追加評価されます:

- **最終判定**: 案件魅力度 × 提案難易度 で最終アクションが決まります
