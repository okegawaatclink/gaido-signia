# QA出力フォーマット定義

質疑一覧の内部JSON形式と、各出力形式（Markdown / Excel）のマッピング定義。

---

## 内部JSON形式（qa_draft.json）

```json
{
  "project_name": "案件名",
  "generated_at": "2025-01-01T00:00:00",
  "source_files": ["RFP_XXX.pdf", "hearing_20250101.docx"],
  "contradictions": [
    {
      "id": "C001",
      "description": "矛盾の概要",
      "file_a": "RFP_XXX.pdf",
      "location_a": "3章",
      "file_b": "hearing_20250101.docx",
      "location_b": "2ページ"
    }
  ],
  "qa_list": [
    {
      "id": "Q001",
      "perspective": "要件曖昧さ",
      "contradiction_id": null,
      "priority": "必須-P1",
      "question": "質疑内容（〜をご教示ください。形式）",
      "background": "質問の背景・目的",
      "reference": "資料名 章番号（p.XX「引用文言」）",
      "status": "未回答",
      "answer": null,
      "reflection_note": null
    }
  ]
}
```

### フィールド定義

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | `Q001` 形式の連番（矛盾は `Q` + 矛盾IDから生成） |
| `perspective` | string | `要件曖昧さ` / `技術的前提` / `評価基準` / `体制・日程` / `矛盾・不整合` |
| `contradiction_id` | string \| null | 矛盾由来の質疑の場合に対応する矛盾ID |
| `priority` | string | `必須-P1` / `必須-P2` / `推奨` |
| `question` | string | 質疑本文（文末は `〜をご教示ください。` または `〜をお聞かせください。` に統一） |
| `background` | string | 質問の背景・目的（1〜2文） |
| `reference` | string | `資料名 章番号（p.XX「引用文言」）` 形式で統一 |
| `status` | string | `未回答` / `回答済み` / `取下げ` |
| `answer` | string \| null | 顧客回答（取込後に設定） |
| `reflection_note` | string \| null | 提案への反映内容（回答解析後に設定） |

### 優先度の定義

| 優先度 | 定義 |
|--------|------|
| `必須-P1` | 未回答では提案書の骨格（構成・アーキテクチャ・費用の大枠）が書けない |
| `必須-P2` | 未回答では費用・体制・リスクの定量化ができない |
| `推奨` | 回答があれば提案品質が向上するが、なくても提案書は成立する |

### 参照箇所の記法

すべての質疑の `reference` フィールドは以下の形式に統一すること:

```
{資料名} {章番号}（p.{ページ番号}「{該当箇所の引用文言（数語）}」）
```

例:
- `RFI 9章（p.15「26年度の画面構築は10画面程度」）`
- `RFI 別紙 データレイク対象ファイル一覧（p.22-25「主なデータレイク対象ファイル」）`
- `RFI 6章（p.10「アプリを窓口にした、顧客に関するデータの活用」）`

ページ番号が不明な場合は章番号と引用文言のみ記載する。

## Markdown出力形式（qa_list.md）

テンプレートなし時の出力。**「一覧テーブル＋詳細セクション」の2ブロック構成**で出力する。

- 一覧テーブルで走査性を確保し、詳細セクションで質疑内容・背景・参照箇所を展開する
- テーブルの「確認事項」列は動詞句による1行要約のみ記載し、セル内に長文を詰め込まない
- 回答収集はExcelで行うため、Markdown出力に回答欄は設けない
- 目次は設けない（観点ごとの `##` 見出しで十分にナビゲーション可能）
- 観点ごとに `##` 見出しで分割する

**重要:** 矛盾検出結果（contradictions.md）は内部分析メモであり、顧客送付用のqa_list.mdには含めない。矛盾由来の質疑はQ番号に `[矛盾 Cxxx]` タグを付与して質疑セクション内に統合する。

```markdown
# 質疑一覧 - {案件名}

生成日時: {日時} ／ 入力資料: {ファイル名一覧}

---

## 回答要領

| 項目 | 内容 |
|------|------|
| P1 回答期限 | {日付を記入してください} |
| P2・推奨 回答期限 | {日付を記入してください} |
| 回答方法 | 添付の回答用Excelファイル（qa_list.xlsx）の「回答」列にご記入ください |
| 送付先 | {担当者名・連絡先} |
| 回答担当者（貴社） | （ご記入ください） |

> **優先度の定義:**
> - **必須-P1（{N1}件）**: 未回答では提案書の構成・アーキテクチャ・費用の大枠が確定できない
> - **必須-P2（{N2}件）**: 未回答では費用・体制・リスクの定量化ができない
> - **推奨（{N3}件）**: 回答があれば提案品質が向上するが、なくても提案書は成立する

---
```

## Excelテンプレート対応マッピング

BoxからダウンロードしたQAテンプレートExcelに出力する際のカラムマッピング。

### テンプレートのカラム構成

| 列 | ヘッダー | 出力内容 | 備考 |
|----|---------|---------|------|
| B | N | `id`（Q001形式） | |
| C | 質問日 | 生成日時（`generated_at` の日付部分） | |
| D | 項目 | `perspective`（観点名）+ 優先度（例: `要件曖昧さ【必須-P1】`） | priorityを括弧内に付記 |
| E | 資料 | `reference` の資料名部分（`p.` より前） | referenceを資料名とページに分割して出力 |
| F | ページ | `reference` のページ番号部分（`p.XX` の数値） | |
| G | 起票者 | 空欄（ユーザーが手動記入） | |
| H | 回答日 | 空欄（顧客が記入） | |
| I | 回答者 | 空欄（顧客が記入） | |
| J | ステータス | `status`（`未回答` / `回答済み` / `取下げ`） | |
| K | 確認事項 | `question` + 改行 + `background` を結合して出力 | backgroundは「【背景】{文}」形式で末尾に付記 |
| L | 回答 | `answer`（取込前は空欄） | |

### マッピングキーワード（自動検出用）

`tools/qa_excel_handler.py` がテンプレートのヘッダー行を自動検出し、以下のキーワードで内部フィールドとマッピングする。

| 内部フィールド | マッピング対象キーワード（部分一致・大小無視） |
|--------------|------------------------------------------|
| `id` | `N`, `No`, `番号`, `ID` |
| `perspective` + `priority` | `項目`, `観点`, `カテゴリ`, `区分` |
| `reference`（資料名） | `資料`, `出典`, `Reference` |
| `reference`（ページ） | `ページ`, `page`, `箇所`, `参照` |
| `status` | `ステータス`, `Status`, `状態` |
| `question` + `background` | `確認事項`, `質疑`, `質問`, `内容`, `Question` |
| `answer` | `回答`, `Answer`, `顧客回答` |
| `generated_at`（日付） | `質問日`, `起票日`, `日付` |

**マッピングできなかったカラム（起票者・回答日・回答者）**: 空列として出力し、ユーザー・顧客が手動記入できるよう残す。

**テンプレートにないフィールド（矛盾IDなど）**: 出力対象外。矛盾由来の質疑は `確認事項` 列の先頭に `[矛盾 Cxxx]` タグを付与して識別する。

## 回答取込形式

顧客が記入した回答済みExcel/Markdownを取り込む際の解析ルール。

### Excel取込

- `L列（回答）` のセル値を `answer` フィールドに設定
- `H列（回答日）` に日付が入っている場合は `status: 回答済み` に変更
- `L列（回答）` が空の場合は `status: 未回答` のまま
- `J列（ステータス）` または `L列（回答）` に「取下げ」「不要」等の記載がある場合は `status: 取下げ` に変更

**注意:** Markdown形式（qa_list.md）は顧客参照用であり、回答収集には使用しない。顧客への回答は必ずExcel（qa_list.xlsx）で収集すること。
