# エッジケース定義テンプレート

エッジケース・異常系を網羅的に定義するためのテンプレート。

---

## 基本ルール

- 最低10件のエッジケースを定義すること
- 境界値と異常系を必ず含めること
- 期待結果を具体的に記載すること

## カテゴリ別の例

### 入力データ系

| ID | テストケース | 入力条件 | 期待結果 |
|----|-------------|----------|----------|
| EDGE-001 | 空データ | 0バイト/空配列 | スキップ、警告ログ |
| EDGE-002 | 上限超過 | サイズ上限を超えたデータ | エラー、警告通知 |
| EDGE-003 | 不正フォーマット | バイナリ、不正エンコーディング | エラーハンドリング |
| EDGE-004 | 特殊文字 | 日本語、絵文字、制御文字 | 正常処理 |
| EDGE-005 | 境界値（下限） | 最小許容値 | 正常処理 |
| EDGE-006 | 境界値（上限） | 最大許容値 | 正常処理 |

### 負荷・並列系

| ID | テストケース | 入力条件 | 期待結果 |
|----|-------------|----------|----------|
| EDGE-007 | 大量データ | 想定最大量の10倍 | バッチ処理、タイムアウトなし |
| EDGE-008 | 同時実行 | 並列リクエスト多発 | rate limit対応 |
| EDGE-009 | 長時間処理 | 処理時間上限ギリギリ | タイムアウト設定通り動作 |

### 外部連携系

| ID | テストケース | 入力条件 | 期待結果 |
|----|-------------|----------|----------|
| EDGE-010 | 外部API障害 | タイムアウト、エラー応答 | リトライ、Degraded Mode |
| EDGE-011 | 認証エラー | 無効なトークン | 処理中断、管理者通知 |
| EDGE-012 | ネットワーク障害 | 接続不可 | リトライ（最大3回） |
| EDGE-013 | 不正応答 | JSONパースエラー | フォールバック出力 |

### セキュリティ系

| ID | テストケース | 入力条件 | 期待結果 |

| EDGE-014 | SQLインジェクション | 悪意あるSQL文字列 | エスケープ、拒否 |

| EDGE-016 | パストラバーサル | ../を含むパス | 拒否 |

### 状態遷移系

| ID | テストケース | 入力条件 | 期待結果 |

| EDGE-017 | 重複実行 | 同一リクエスト2回 | 冪等性保証 |
