# 品質チェックリスト

成果物の完成度を検証するためのチェックリスト。

---

## 使い方

1. 成果物の種類に応じたセクションを使用
2. 全ての `[必須]` 項目が満たされていることを確認
3. 確認結果を記録

---

## 設計書の品質チェック

```markdown
### 必須項目

- [ ] [必須] コスト見積もりが含まれている
- [ ] [必須] 並列処理設計（並列度、バッチサイズ）が明記されている
- [ ] [必須] エッジケース・異常系が10件以上定義されている
- [ ] [必須] 外部サービス障害時の縮退設計が記載されている
- [ ] [必須] 品質ゲートの数値基準が明記されている
- [ ] [必須] 参照元ドキュメントが明記されている
- [ ] [必須] 成果物と終了条件が分離されている

### 推奨項目

- [ ] [推奨] 技術選定の根拠が記載されている
- [ ] [推奨] バージョン管理戦略が明記されている
- [ ] [推奨] ロールバック手順が記載されている

### 任意項目

- [ ] [任意] キャッシュ戦略が記載されている
- [ ] [任意] パフォーマンスベンチマーク結果がある
```

---

## タスク分解の品質チェック

```markdown
### 必須項目

- [ ] [必須] 全タスクに受入条件が記載されている
- [ ] [必須] 受入条件に曖昧な表現がない
- [ ] [必須] 回帰テストマーキング（○/△/×）がある
- [ ] [必須] 品質分類タグ（[必須]/[推奨]/[任意]）がある

### 推奨項目

- [ ] [推奨] サンプルデータと検証観点のマッピングがある
- [ ] [推奨] good例・bad例の検証方法が明記されている
- [ ] [推奨] 見積もりが記載されている

### 任意項目

- [ ] [任意] CI/CDフェーズ指定がある
- [ ] [任意] 依存関係図がある
```

---

## テスト計画の品質チェック

```markdown
### 必須項目

- [ ] [必須] カバレッジ基準が数値で定義されている
    - Line Coverage: ≥80%
    - Branch Coverage: ≥70%
    - Function Coverage: ≥80%
- [ ] [必須] エッジケースが境界値と異常系を含んでいる
- [ ] [必須] 期待結果が具体的に記載されている

### 推奨項目

- [ ] [推奨] good例の検証方法が明記されている
    - 例: 「重大な指摘（error/warning）がないこと」
- [ ] [推奨] 自動化可能なテストが特定されている
- [ ] [推奨] テストデータの準備方法が記載されている

### 任意項目

- [ ] [任意] 性能テストシナリオがある
- [ ] [任意] セキュリティテスト計画がある
```

---

## Epic/PBI の品質チェック

```markdown
### 必須項目

- [ ] [必須] ユーザーストーリー形式になっている
- [ ] [必須] INVESTチェックが完了している
- [ ] [必須] YAGNIチェックが完了している
- [ ] [必須] 受入条件が検証可能な形で記載されている

### PBI追加項目

- [ ] [必須] 完成の定義（DoD）が明記されている
- [ ] [必須] 品質分類タグが付与されている

### 推奨項目

- [ ] [推奨] 依存関係が明記されている
- [ ] [推奨] リスクが特定されている
```
