# 縮退設計（Degraded Mode）テンプレート

外部サービス障害時の動作を定義するためのテンプレート。

---

## 基本原則

- 外部サービス障害時も、可能な限りサービスを継続する
- 完全停止ではなく、機能制限で継続する
- ユーザーに状態を明示する

---

## 縮退フロー

```
外部サービスへのリクエスト
    │
    ▼
正常応答？ ───────── Yes ──→ 通常処理
    │
    No（タイムアウト/エラー）
    │
    ▼
リトライ（最大N回）
    │
    ▼
復旧？ ─────────── Yes ──→ 通常処理
    │
    No
    │
    ▼
縮退モード移行
    ├─ 警告ログ出力
    ├─ 監視アラート発報
    └─ 機能制限で継続
```

---

## テンプレート

```markdown
## 縮退設計

### サービス別の縮退動作

| サービス | 正常時の動作 | 障害時の動作 | 影響範囲 |
|----------|-------------|-------------|----------|
| | | | |
| | | | |

### リトライ設定

| サービス | リトライ回数 | リトライ間隔 | タイムアウト |
|----------|-------------|-------------|-------------|
| | | | |

### アラート設定

| サービス | 検知条件 | 通知先 | 重要度 |
|----------|----------|--------|--------|
| | | | |

### 復旧手順

| サービス | 復旧確認方法 | 自動復旧 |
|----------|-------------|----------|
| | | Yes/No |
```

## 記入例

```markdown
## 縮退設計：ECサイト

### サービス別の縮退動作

| サービス | 正常時の動作 | 障害時の動作 | 影響範囲 |
|----------|-------------|-------------|----------|
| 決済API | リアルタイム決済 | 決済一時停止、後日再試行キュー | 購入完了不可 |
| 在庫API | リアルタイム在庫チェック | キャッシュ値を使用（5分前） | 在庫切れ誤表示の可能性 |
| レコメンドAPI | パーソナライズ推薦 | 人気商品を静的表示 | 推薦精度低下 |
| 画像CDN | 高解像度画像配信 | 低解像度フォールバック | 表示品質低下 |
| 検索API | リアルタイム検索 | キャッシュ検索結果 | 最新商品未反映 |

### リトライ設定

| サービス | リトライ回数 | リトライ間隔 | タイムアウト |
|----------|-------------|-------------|-------------|
| 決済API | 3回 | 1s, 2s, 4s（指数バックオフ） | 30s |
| 在庫API | 2回 | 500ms | 5s |
| レコメンドAPI | 1回 | 500ms | 3s |
| 画像CDN | 2回 | 1s | 10s |
| 検索API | 2回 | 500ms | 5s |

### アラート設定

| サービス | 検知条件 | 通知先 | 重要度 |
|----------|----------|--------|--------|
| 決済API | 3回連続失敗 | PagerDuty + Slack | Critical |
| 在庫API | 5分間エラー率>10% | Slack | Warning |
| レコメンドAPI | エラー率>50% | Slack | Info |
| 画像CDN | 1分間エラー率>30% | Slack | Warning |

### 復旧手順

| サービス | 復旧確認方法 | 自動復旧 |
|----------|-------------|----------|
| 決済API | ヘルスチェックエンドポイント | Yes（5分間隔でチェック） |
| 在庫API | 正常応答3回連続 | Yes |
| レコメンドAPI | 正常応答1回 | Yes |
```

## 縮退モードの通知

```markdown
## ユーザー向け表示

### 決済サービス障害時
「現在、決済処理に時間がかかっております。
しばらくお待ちいただくか、後ほど再度お試しください。」

### 検索サービス障害時
「検索機能が一時的に制限されています。
カテゴリからの商品閲覧は可能です。」
```

---

## 成果物への注記

縮退モードで実行された場合、成果物に以下を注記：

```markdown
⚠️ **注意**: この処理は縮退モードで実行されました。
- 障害サービス: {サービス名}
- 影響: {具体的な影響}
- 実行日時: {timestamp}
```
