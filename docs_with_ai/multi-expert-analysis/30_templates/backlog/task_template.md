## 記入例

```markdown
# Task 1.2.1: パスワードリセットAPIを実装する

## 親PBI

#2 PBI 1.2: パスワードを忘れても復旧できる

## 作業内容

- POST /api/password-reset エンドポイント作成
- リセットトークン生成ロジック実装（SHA256、有効期限24h）
- トークンのDB保存処理実装
- メール送信サービス呼び出し実装

## 完了条件

- [ ] エンドポイントが正常に動作する
- [ ] ユニットテストが通る（カバレッジ80%以上）
- [ ] コードレビュー完了
- [ ] OpenAPI仕様書に追記

## 技術的な注意点

- トークンは推測不可能なものを使用（UUID v4 + SHA256）
- レート制限を設ける（同一メールに対して1時間に3回まで）
- タイミング攻撃対策（存在しないメールでも同じレスポンス時間）

## 依存関係

- **前提**: メール送信サービスのAPIが利用可能
- **ブロッカー**: なし

## 参考資料

- OWASP パスワードリセットガイドライン
- 既存の認証API実装（/api/auth/login）
```

## タイトルの書き方

### 番号ルール

```
Task [Epic番号].[PBI連番].[Task連番]: [動詞で始まる具体的作業]
```

- **Epic番号**: 親EpicのEpic番号
- **PBI連番**: 親PBIのPBI連番
- **Task連番**: 依存関係順（依存先より大きい番号を付ける）

**重要**: Task番号順 = 実装順。依存先Taskより小さい番号を付けてはならない。

### 良い例

```
Task 1.1.1: データベーススキーマを作成する（依存なし）
Task 1.1.2: 認証APIのエンドポイントを実装する（1.1.1に依存）
Task 1.1.3: ログイン画面のUIコンポーネントを作成する（1.1.2に依存）
Task 1.1.4: 単体テストを追加する（1.1.2に依存）
```

### 悪い例

```
× Task 1.1.1: API  ← 動詞がない、何のAPIか不明
× Task 1.1.1: 開発する  ← 何を開発するか不明
× Task: その他  ← 番号がない、曖昧すぎる
× Task 1.1.1: バグ修正  ← どのバグか不明
```
