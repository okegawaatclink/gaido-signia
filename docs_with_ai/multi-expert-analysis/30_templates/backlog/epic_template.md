-

## 成功指標（KPI）

| 指標 | 目標値 | 計測方法 |
|------|--------|----------|
| | | |

## INVESTチェック

- [ ] **I**ndependent: 他Epicと独立しているか
- [ ] **N**egotiable: 交渉・調整の余地があるか
- [ ] **V**aluable: ビジネス価値があるか
- [ ] **E**stimable: 見積もり可能か
- [ ] **S**mall: 適切なサイズか
- [ ] **T**estable: 検証可能か

## YAGNIチェック

- [ ] 今この瞬間、これがないと先に進めないか？
- [ ] 作らないことで10倍以上の手戻りが発生するか？
- [ ] 推測ではなく実際の要求に基づいているか？

## 記入例

```markdown
# Epic 1: ユーザーが安全にログインできる

## ユーザーストーリー

**As a** 一般ユーザー
**I want to** 安全にシステムにログインしたい
**So that** 自分のデータを保護しながらサービスを利用できる

## ビジネス価値

- セキュリティ要件の充足
- ユーザー信頼の獲得
- コンプライアンス対応

## 成功指標（KPI）

| 指標 | 目標値 | 計測方法 |
|------|--------|----------|
| ログイン成功率 | 99%以上 | ログ集計 |
| セキュリティインシデント | 0件 | 監視アラート |

## INVESTチェック

- [x] **I**ndependent: 他Epicと独立しているか
- [x] **N**egotiable: 交渉・調整の余地があるか
- [x] **V**aluable: ビジネス価値があるか
- [x] **E**stimable: 見積もり可能か
- [x] **S**mall: 適切なサイズか
- [x] **T**estable: 検証可能か

## YAGNIチェック

- [x] 今この瞬間、これがないと先に進めないか？ → Yes、認証なしでは開始できない
- [x] 作らないことで10倍以上の手戻りが発生するか？ → Yes
- [x] 推測ではなく実際の要求に基づいているか？ → Yes、要件定義書に記載

## 含まれるPBI

- [ ] PBI 1.1: ログインできる
- [ ] PBI 1.2: パスワードを忘れても復旧できる
- [ ] PBI 1.3: ログアウトできる
```
