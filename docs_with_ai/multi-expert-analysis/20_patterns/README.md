# パターン選択ガイド

案件タイプに応じた分析パターンを選択します。

---

## パターン一覧

| パターン | ファイル | 用途 |
|----------|----------|------|
| 新規開発 | `new_development.md` | ゼロからシステム/機能を構築 |
| 保守・改修 | `maintenance.md` | 既存システムの改修・機能追加 |
| データ基盤 | `data_pipeline.md` | ETL/ELT、データパイプライン構築 |

## パターン別の特徴

| 観点 | new_development | maintenance | data_pipeline |
|------|-----------------|-------------|---------------|
| **主な関心** | ゼロから作る | 既存を壊さない | データ品質・リネージ |
| **Architectの焦点** | 技術選定、構造設計 | 影響範囲、後方互換 | レイヤー設計、スキーマ |
| **QAの焦点** | テスト戦略構築 | 回帰テスト | データ検証 |
| **Securityの焦点** | 設計段階から組込み | 既存の穴を塞ぐ | データ保護、アクセス制御 |
| **追加専門家** | - | - | Data Architect（必須） |

## 複数パターンの適用

複数のパターンが当てはまる場合は、主パターンを選択し、
サブパターンの観点を追加で考慮します。

**例：既存システムにデータ基盤を追加**
- 主パターン: `maintenance`
- サブパターン: `data_pipeline` の観点を追加
- 専門家: core + data_architect

```yaml
# 10_project/context.md での指定例
project:
  type: maintenance
  sub_patterns:
    - data_pipeline

experts:
  core: all
  optional:
    - data_architect
```
